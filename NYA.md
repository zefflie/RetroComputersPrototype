# Документация ассемблера .nya
Подразумевается наличие базовых знаний о языках ассемблера. Если нет, то [поисковик](https://duckduckgo.com) и [нейронка](https://duck.ai) в помощь :> <br>
[Здесь](https://kaf401.rloc.ru/DSP/I8080_commands.htm) есть описание всех инструкций i8080. 

## Препроцессинг
Игнорируются комментарии (начинаются на `;` и заканчиваются в конце строки).<br>
Игнорируются отступы в начале и конце строки.

## Формат команды
```
[<имя метки>:] <мнемоника> [операнд 1][... , <операнд N>]
```

## Операнды
- Регистры `b`, `c`, `d`, `e`, `h`, `l`, `m` (особенности реализации, это обращение к памяти по адресу регистра hl), `a`
- Парные регистры `bc`, `de`, `hl`, `psw` (флаговый регистр и регистр a)
- Числовой литерал
  - Десятичный `0` `20` `40`
  - Двоичный `0b0` `0b1100` `0b00110010`
  - Шестнадцатеричный `0x0` `0x4F` `0x7E1A`
- Символьный литерал `'0'` `'H'` `\r` (поддерживаются спецсимволы `\0` `\n` `\r` `\t` `\v` `\f` `\b`, и Escape-экранирование `\'` `\"` `\\`)
- Строчный литерал `"любой текст"` (поддерживает спецсимволы и экранирование, поддерживается только директивой `db`)
- Имя метки (разыменовывется в литерал со своим значением)

## Директивы

### EQU (equal)
```
метка: equ <литерал>
```
Присваивает предшествующей метке конкретное значение

### DB (define byte)
```
db [... , литералы]
```
Сохраняет операнды как последовательность байт в конечном файле. Количество операндов неограниченно.

### SKIP
```
skip <литерал>
```
Пропускает указанное литералом количество байт в конечном файле, заполняя их нулями.

## Примечания
Отсутствует препроцессор выражений.<br>
Здесь нет синтаксиса [] для адресации в память.<br>
В отличие от других ассемблеров пары регистров указываются не именем старшего регистра а именами обоих разом. т.е. не `b` а `bc`, не `h` а `hl`, это не касается `psw`.<br>
